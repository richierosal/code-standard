## Test Strategy Standard

Every feature must define:

* Unit Test Scope
* Integration Test Scope
* API Test Scope
* UI Test Scope
* E2E Test Scope
* Regression Scope
* Performance Scope

## Risk Based Priority

Prioritize tests by:

1. Business Criticality
2. User Impact
3. Technical Complexity
4. Security Exposure
5. Production Risk

## Minimum Coverage

Required:

* Happy Path
* Negative Path
* Edge Case
* Error Handling
* Permission Scenario
* Regression Impact
