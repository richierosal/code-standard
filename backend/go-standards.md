# Go Standards

## Architecture

Preferred:

* Clean Architecture

Layers:

* Domain
* Usecase
* Repository
* Delivery
* Infrastructure

## Dependency Injection

Required

Preferred:

* Constructor Injection

## Logging

Structured Logging Only

## Error Handling

* Explicit Errors
* Wrapped Errors

## Testing

Required:

* Unit Test
* Integration Test

## Package Rules

Small Packages

Single Responsibility

Avoid cyclic dependency.
